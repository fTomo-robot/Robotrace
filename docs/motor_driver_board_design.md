# ブラシDCモータ制御基板 設計ドキュメント

- **主要IC**: STM32F469VGT6 (MCU) / ICM-42688-P (6軸IMU) / MIC4606-2 ×2 (フルブリッジ・ゲートドライバ, 1個=1モータ分, PWM入力方式)
- **電源**: 4S LiPo (公称14.8V, フル充電16.8V, カットオフ想定12–13.2V)
- **モータ**: maxon DCX10L ×2台 (10mm径ブラシDCモータ, 巻線仕様は未確定 — 拘束電流は巻線により0.44〜4.23A/台まで変動)
- **用途**: 上記モータ2台をそれぞれ独立したフルHブリッジで駆動し、IMUで姿勢/振動をフィードバックするロボット用モータドライバボード
- **リビジョン**: Rev.E (draft) / 2026-07-05 — MIC4606-1からMIC4606-2(単一PWM入力方式)に変更しTIM1のみで2モータ分を賄えるように。さらにMOSFETをNexperia BUK7K13-60E(デュアル品、1パッケージ=1レッグ)に確定し部品点数を半減

> 本ドキュメントはスキーマ設計に入る前のアーキテクチャ設計書です。ピン番号レベルの割当は STM32CubeMX の
> ピンアウトツールで最終確認し、各ICの正式データシートで定格・パッケージ・脚番号を照合してから回路図に落としてください。

> **Rev.D変更点**: MIC4606-2はレッグごとに1本のPWM入力(APWM/BPWM)で高速側/低速側を自動的に相補駆動する(データシート Table 7-2 で確認済み)。これにより、MIC4606-1で必要だった4本の独立入力(AHI/ALI/BHI/BLI)が、レッグあたり1本・1チップあたり2本(APWM/BPWM)に半減。**TIM1の1個の相補チャネルペア(CHx/CHxN)だけでモータ1台分のフルブリッジを駆動できる**ため、モータ2台分をTIM1だけで賄えるようになり、Rev.Cで実ピン未確認だったTIM8が不要になった。

**設計方針**

- ゲートドライブ用10-12Vレールとロジック用3.3V/5Vレールを分離し、MIC4606のVDD (5.5–16V) を確保しつつMCU/IMUには3.3Vを供給する。
- モータ電流経路 (VBAT→Hブリッジ→モータ) はロジック電源を経由させず、バッテリーから直接引く。
- **モータ2台は完全に独立した2系統**(それぞれ専用のMIC4606-2・4石MOSFET・シャント・エンコーダ)として構成し、片方の故障が他方に波及しないようにする。
- **駆動方式はロックドアンチフェーズ(bipolar)方式**: TIM1の相補チャネル(CHx/CHxN)をそのままAPWM/BPWMに接続し、デューティ50%で停止、50%超/未満で正転/逆転を表現する(詳細は3.2節)。
- ハードウェアフォルト(過電流/過熱)は各モータのMIC4606のENピンを独立に引き下げることで実現する(EN=Highで動作、Lowでシャットダウン)。TIM1のBKINは両モータ共有のため、個別の瞬時遮断はENピン側で行う(詳細は3.4節)。

> **重要**: 4S LiPoはフル充電で16.8Vに達し、MIC4606のVDD推奨上限(16V)を超える。バッテリーをゲートドライバVDDに直結してはならず、必ず10-12V程度に落とす降圧段(LDOで可、詳細は次項)を挟むこと。

## 2. 電源アーキテクチャ

| レール | 用途 | 生成方法 | 想定電流 | 備考 |
|---|---|---|---|---|
| VBAT (4S LiPo) | Hブリッジ電源、降圧段入力 | バッテリー直結, 10.5–16.8V | 〜4A級/モータ (最悪同時拘束で〜8.5A) | 逆接保護・入力ヒューズ・バルクコン必須。ヒューズ/コネクタは2台同時拘束の瞬間最大値も考慮 |
| 10-12V (VGATE) | MIC4606-2 ×2 の VDD | VBAT→10-12V リニアレギュレータ(LDO) | 〜150mA (2チップ分) | 電流が小さいためBuckでなくLDOで十分。**VBATを直結しないこと(VDD上限16V超過)** |

**保護回路**

- 入力: 逆接保護 (Pch MOSFETによる理想ダイオード、またはショットキー)、ポリスイッチ/ヒューズ、TVSサージ保護 (4S LiPoの過充電/回生による瞬間的な電圧上昇も考慮しマージンを持たせる)
- VBAT/VGATEバルクコンデンサ (低ESR, ≥25V耐圧) を各Hブリッジ直近に配置しスイッチング時の電圧リンギングを抑制
- 10-12V, 5V, 3.3V各レールにUVLO/シーケンスを考慮 (MCUリセットはPOR ICまたは内蔵POR + 外付けRCで十分)

## 3. モータパワー段 (独立フルHブリッジ ×2, MIC4606-2 ×2)

### 3.1 構成

MIC4606は16pin 1パッケージの中に **Phase A側・Phase B側の両方のハーフブリッジドライバ**を内蔵しており、**1個で1台分のフルHブリッジ(4石)を完結**できる(データシート Table 4-2 で確認済み)。モータ2台を使うため、MIC4606-2を1個ずつ、完全に独立した系統として割り当てる。

- **MIC4606-2 #1** → DCX10L #1 用フルブリッジ (Leg A: Q1ハイサイド/Q2ローサイド, Leg B: Q3ハイサイド/Q4ローサイド)
- **MIC4606-2 #2** → DCX10L #2 用フルブリッジ (Leg A: Q5ハイサイド/Q6ローサイド, Leg B: Q7ハイサイド/Q8ローサイド)
- 各モータは、対応するチップのAHS(Leg A中点)-BHS(Leg B中点)間に接続 → 正逆転・PWM速度制御が可能なフルブリッジ
- **型番はMIC4606-2**(レッグごとに1本のPWM入力 APWM/BPWM で高速側/低速側を自動相補駆動する品番。MIC4606-1の4本独立入力(AHI/ALI/BHI/BLI)より配線がシンプル)

### 3.2 MIC4606-2 の駆動方式 (PWM入力モード)

データシート Table 7-2 (MIC4606-2 Truth Table) で確認した動作:

| xPWM | xLO | xHO | 備考 |
|---|---|---|---|
| 0 | 1 | 0 | ローサイドON (出力がGND側) |
| 1 | 0 | 1 | ハイサイドON (出力がVBAT側、xLOが1.9V未満に落ちてから遷移) |

デッドタイム(ハイ/ロー切替の空白期間)はMIC4606-2内部で自動生成されるため、外部からは1本のPWMを与えるだけでよい。

**推奨駆動方式: ロックドアンチフェーズ(bipolar)方式**

- Leg AのPWM(APWM)にTIM1のメインチャンネル(例: CH1)、Leg BのPWM(BPWM)にその**相補チャンネル(CH1N)**をそのまま接続する。CH1Nは既にCH1の論理反転(+デッドタイムギャップ)なので、追加のロジックなしでAPWM/BPWMの関係が作れる。
- デューティ D (CH1のデューティ) に対し、モータ電圧 ≈ (2D − 1) × VBAT となる。**D=50%で停止、D>50%で正転、D<50%で逆転**。
- 利点: MCU側は1個の相補チャネルペアだけで1モータ分のフルブリッジを駆動できる (TIM1 1個で2モータ分が賄える)。低速域でのデッドゾーンが生じにくく、回生制動も自然に行える。
- 常に両レッグがスイッチングするため、片側を固定して駆動する「符号-絶対値(sign-magnitude)」方式より若干スイッチング損失は増えるが、この電流レベル(数A級)では無視できる。

### 3.3 MIC4606-2 ピン接続まとめ (1個あたり)

データシート(DS20005604D, Table 4-2)で全ピンを確認済み。MIC4606-1との違いはピン13-16(BPWM/NC/NC/APWM)のみで、他は共通。

| ピン | 接続先 |
|---|---|
| VDD | 10-12Vゲートドライブレール。VSSとの間に>1µFデカップリング |
| VSS | パワーグランド(ドライバ基準)。QFN品はEPもここに接続 |
| EN | MCU GPIO (Highで動作/Lowでシャットダウン)。**フローティング禁止**。過電流ラッチ出力とワイヤードANDし、フォルト時はLowに引く |
| APWM | Leg A用PWM入力 ← MCU TIM1 CHx (メインチャンネル) |
| BPWM | Leg B用PWM入力 ← MCU TIM1 CHxN (相補チャンネル) |
| AHO | Leg A ハイサイドMOSFETゲートへ (ゲート抵抗経由) |
| ALO | Leg A ローサイドMOSFETゲートへ (ゲート抵抗経由) |
| AHB | Leg Aブートストラップ。AHB-AHS間に1µFコンデンサ (25V以上耐圧) |
| AHS | Leg Aスイッチノード → モータ出力の一端 |
| BHO | Leg B ハイサイドMOSFETゲートへ |
| BLO | Leg B ローサイドMOSFETゲートへ |
| BHB | Leg Bブートストラップ。BHB-BHS間に1µFコンデンサ |
| BHS | Leg Bスイッチノード → モータ出力の他端 |
| NC (×3) | 未接続 |
| EP (QFN露出パッド) | VSSに接続、放熱のためグランドプレーンへ |

- 入力しきい値は **VIL(max)=0.8V / VIH(min)=2.2V の固定TTLレベル**(VDDに関係なく一定)とデータシートで確認済み。→ **3.3V MCU直結で問題なし**(レベルシフタ不要)
- ゲート抵抗: HO/LOそれぞれに4.7〜10Ω程度を挿入 (ターンオン/ターンオフを非対称にしたい場合はダイオード並列で分離)
- **HSピンクランプ推奨(データシート7.2節)**: AHS/BHSピンとモータ相ノード間に抵抗+ダイオードのクランプを入れ、デッドタイム中のボディダイオード導通による負電圧グリッチからピンを保護する

### 3.4 MOSFET選定方針 — `Nexperia BUK7K13-60E` を採用 (デュアルMOSFET×4個, 2モータ分)

DCX10Lは10mm径の精密小型モータで、公称電流は巻線仕様によらず1A未満、拘束電流も巻線により0.44A(12V巻線)〜4.23A(1.5V巻線)/台まで変動する。

**採用: `Nexperia BUK7K13-60E`** — Nexperia公式データシート(2015-09-02版)で仕様・ピン構成を確認済み。

| 項目 | 値 |
|---|---|
| 構成 | **デュアルNchMOSFET、FET1/FET2は完全に独立**(共通ドレイン/ソースではなく、S1/G1/D1とS2/G2/D2が別々) |
| パッケージ | LFPAK56D (SOT1205), 8リード, "Dual Power-SO8" |
| VDS | 60V |
| RDS(on) | 8mΩ typ / 10mΩ max @ Vgs=10V, ID=10A |
| ID | 40A (Tmb=25℃, パッケージ律速) / 38A (100℃) |
| QGD | 9.7nC (Vgs=10V, Vds=48V) |
| Vgs(th) | >1V @175℃ (標準しきい値品、MIC4606の10-12Vゲート駆動に好適) |
| 認証/用途 | AEC-Q101車載品質。データシート記載の用途例に "Motors, lamps and solenoid control" を明記 — モータ駆動を想定した部品 |

**ピン配置** (データシート Table 2): 1=S1, 2=G1, 3=S2, 4=G2, 5=D2, 6=D2, 7=D1, 8=D1。FET1(S1/G1/D1)とFET2(S2/G2/D2)は内部で繋がっておらず**完全に独立した2石**のため、**1パッケージ = 1レッグ分(ハイサイド+ローサイド)としてそのまま使える**。

- **採用理由**: VDS=60Vは4S LiPoフル充電16.8Vに対し約3.6倍のマージンがあり、以前検討した40V級品より余裕が大きい。RDS(on)=8-10mΩはDCX10Lの電流レベル(最大4.23A/台)に対し十分低損失(4Aで約0.16W/石、ヒートシンク不要)。何よりデュアル構成により**部品点数を半減**できる: 1レッグ=1パッケージなので、1モータ(2レッグ)=2パッケージ、モータ2台で**合計4パッケージ(Q1/Q2=1個目, Q3/Q4=2個目, Q5/Q6=3個目, Q7/Q8=4個目)**、実装点数がBSC016N04LSG単体品×8個の場合の半分になる。
- ロックドアンチフェーズ駆動は常に両レッグがスイッチングするため、ゲート駆動損失・スイッチング損失がわずかに増える点を考慮(QGD9.7nC・この電流レベルでは無視できる範囲)
- ヒートシンク・大型パッケージ(TO220/D2PAK等)は本用途では不要

### 3.5 電流検出・保護 (モータ2台分, 各系統独立)

- 各モータのローサイド共通リターンにシャント抵抗 (**20–50mΩ, 0.25–0.5W**) を挿入し、双方向電流検出アンプ (例: TI INA240 / INA180系 ×2) でMCU ADCへ
- デュアルコンパレータ (例: TLV3202 等) で各モータのシャント電圧と基準電圧を比較し、過電流時に**対応するMIC4606のENへ即時ラッチ**(SR latchでソフト介入なしに保持、MCU側でクリア)。**2系統は互いに独立**しており、片方の過電流がもう片方を止めない
- TIM1のBKINは2モータで共有のため、個別モータの瞬時遮断には使わず、**両モータ同時の緊急停止(手動E-Stopボタン等)用の共通ブレーク入力**として位置づける。モータ毎の独立ハードウェア遮断は上記のEN経由で行う
- 各MOSFET群近傍にNTCサーミスタを配置し温度監視 (ADC, モータ毎に1個)

## 4. STM32F469VGT6 まわり

### 4.1 基本回路

- HSE: 8MHz水晶 + 負荷容量 (OSC_IN/OSC_OUT)
- リセット: NRSTにRCまたは専用スーパーバイザIC
- BOOT0: プルダウン + DFU起動用ジャンパ/ヘッダ
- SWDデバッグヘッダ: **STDC14コネクタ** (ST-Link V3で使われる14ピン, 1.27mmピッチ 2×7) を採用。14ピンのST-Link(V3MINIE等)を使う想定に合わせ、汎用6ピンではなくST純正のSTDC14フットプリントで実装する
  - ピン3–12がARM標準10ピンCortex Debugコネクタと同一構成 (VTref/SWDIO/GND/SWCLK/GND/SWO/Key/NC/GNDDetect/NRST)、ピン13–14がUART (VCP: TX/RX)、ピン1–2は予約。**正式なピン配置は ST AN4989 / UM2448 で最終確認すること**
  - VTref: ST-Link側の電圧センス用 (ST-Linkから給電させず、本ボードは自己給電が前提。誤って両電源が競合しないよう基本はセンスのみとする)
  - SWO (PB3): ITM/printfトレース用
  - **UART (ピン13–14)**: STDC14のVCP用UARTを、既存のUSART2デバッグコンソール(PA2/PA3)にそのまま配線すれば、ST-Link V3のUSB仮想COMポート経由でデバッグログを見られる
- VDDA/VREF+ 用のフェライトビーズ+デカップリングでアナログ電源分離
- **USB OTG FSはDevice専用として実装**(Host/OTGデュアルロールは不要)。IDピン(PA10)を物理コネクタに接続せず、その分をTIM1_CH3に転用する
- **TIM8はモータ駆動に使わないため未使用・将来拡張(3台目のモータやPWM出力等)用に温存**

## 5. ICM-42688-P (IMU) インタフェース

- インタフェース: SPIモード0/3, 最大24MHz対応 (推奨: 8–12MHz程度で開始し実機でマージン確認)
- 電源: VDD/VDDIOともに3.3V系 (共通3.3Vレールから供給可、範囲1.71–3.6V)
- デカップリング: VDD/VDDIO直近に100nF、必要に応じ1µFバルク
- INT1をデータレディ割り込みとしてEXTI接続、制御ループ (姿勢推定/振動補償) を割り込み同期で駆動
- 配置: モータ/Hブリッジのスイッチングノイズ・磁気干渉を避けるため、パワー段およびモータ本体から離し、基板上でMCU寄りかつグランド分離された領域に配置。基板上に軸方向のシルク表記を入れる

## 9. 基板レイアウト指針 (次工程向けメモ)

- パワー段グランド (2系統のHブリッジ/シャント/モータリターン) とロジックグランドを分離し、各シャント抵抗付近の1点でスターグランド接続
- Motor1系統とMotor2系統はできるだけ物理的にも分離し、クロストークや熱的な影響を抑える
- HO/LOからゲートまでの配線はできるだけ短く低インダクタンスに、ゲート抵抗はMOSFETゲート直近配置
- モータ電流経路(最大4A級/台)は一般的な信号/電源パターン幅で十分だが、拘束電流を考慮し余裕を持たせる (IPC-2221目安で確認)
- ブートストラップコンデンサはHB-HSピン直近に配置 (各チップ2箇所ずつ、計4箇所)
- AHS/BHSピンとモータ相ノード間のHSクランプ(抵抗+ダイオード)も各チップ2箇所ずつ配置
- 各ICのデカップリングコンデンサはピン直近配置を徹底
- IMUはモータ/パワー段からできるだけ離し、磁気干渉・振動伝搬・熱伝導を避ける(2系統分のノイズ源から離す)
